## 1. Chosen Scenario
**Scenario:** Local Small Business Digital Presence.  
My project is a website for a local flower shop called "Flower World". It allows users to explore floral collections, check pricing and delivery options, and request a custom bouquet order.

## 2. Implemented Requirements Checklist
I have successfully implemented the following technical requirements:
- [x] **Structure:** Created exactly 4 pages (`index.html`, `about.html`, `order.html`, `pricing.html`).
- [x] **Semantic HTML:** Used semantic landmarks (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`) and a proper heading hierarchy with one `<h1>` per page.
- [x] **Navigation:** The same menu is present on all pages, allowing users to navigate to any page and back to the homepage.
- [x] **Accessibility:** Added a "Skip to main content" link on every page. The form uses explicit `<label>`, `<fieldset>`, and `<legend>` tags.
- [x] **Table:** The `pricing.html` page includes a proper data table with 4 data rows, a `<caption>`, `<th>` elements, and `scope` attributes.
- [x] **Form:** The `order.html` page features an accessible form with 5 controls (text input, email, select, textarea, and submit button).
- [x] **CSS:** Used one external stylesheet (`styles.css`) for consistent styling, typography, spacing, and visual hierarchy.
- [x] **Validation:** All HTML files and the CSS file pass W3C validation with zero errors.

## 3. Known Limitations
Current limitations of this project include:
- The order form is purely for demonstration purposes and does not submit real data to a backend server (`action="#"`).
- Images use `object-fit: cover` with fixed dimensions to maintain a consistent layout, which might result in slight cropping of the original photos.
